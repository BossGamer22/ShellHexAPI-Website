    Codes Examples

    Script_SelfProtect

         
    Script_SelfProtect Code Example, It will make the script Impossible to Disable and Change the Current Path of the Script.
    Arguments Meanings:
    ScriptPath Means is Where is the Script Path you want for the Protection to work.
    ScriptParentClass Means is The Script Current Parent ClassName so if the Parent ClassName is not the Same it will Trigger the Code.
    CanFireRemote Means is if the boolean is true it will fire the Remote then the Server Script will be Triggered,
    which is in the Module Resources Folder With the Function Name.
    
    
    --Variables
local ShellHex = require(workspace.ShellHex)

--Main
ShellHex.Script_SelfProtect(script, "Model", false)

    Script_AntiDeletion

         
    Script_AntiDeletion Code Example, It will make the script Impossible to Delete.
    Arguments Meanings:
    ScriptParentPath Means the Path of the script you want to Protect.
    ScriptName Means is the Name of the Script you want to Protect on the Path.
    
    
    --Variables
local Players = game:GetService('Players')
local ShellHex = require(workspace.ShellHex)

--Main
Players.PlayerAdded:Connect(function(player)
	player.CharacterAdded:Connect(function(character)
		ShellHex.Script_AntiDeletion(character, "AntiExploit") --The Script i want to Protect is Named AntiExploit,
	end) --And Location is in the Player Character.
end)

    Self_GetPlayerScriptCountStatus

         
    Self_GetPlayerScriptCountStatus Code Example, It Return the Player Self Scripts Error Count,Warn Count,Print Count Status.
